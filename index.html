Part 1: HTML Basics

UNDERSTANDING THE WEB

You access the Internet via a browser on your computer (e.g., Chrome, Firefox, etc.). The Internet protocol, HTTP, manages how data is transferred from your browser to the Internet and then back to your computer. The Internet is comprised of millions of servers which communicate and send back (via HTTP) data based on what you're doing via your browser. Most of what you see on the Internet are HTML files.

BEGINNING HTML

HTML is the language of the Internet. The HTML language is comprised of tags "< >" and elements. (e.g., title, body, h1). Most HTML tags have both an "opening" tag "< >" and a "closing" tag "</ >". The content you want to effect with the tag should be written between the "opening" and "closing" tag(s). HTML tags that do not "close" (i.e., do not have a "closing" tag) are called void tags.

Every HTML page must begin with certain "system information". This information is read by the browser and must be stored in the HTML "header". HTML "header" information is everything above the opening body tag. Everything between the "opening" body tag and the "closing body tag is what will appear on your webpage.

"INLINE" & "BLOCK" ELEMENTS

Inline elements (such as the "anchor", "line break", "image", and "span" tags) are HTML tags that do not affect the area around the text. It treats the text like other text on the same line. Inline tags (like "span") will keep text enclosed in the tags on the same line as the surrounding text.

Block elements (such as the "paragraph", "form", and "div" tags) are HTML tags that do affect the area around the text by essentially putting an invisible box around the text, allowing you to set the specific width and/or height of the box

.
Block tags will push text enclosed in the tags to a new line.

Part 2: Creating Structure with HTML

MORE ON "INLINE" & "BLOCK" ELEMENTS

Using Inline and Block tags/elements can help to add structure to your webpage. Before beginning any webpage design, you should begin with a "sketch" or "mock-up" of how the page should look. Irregardless of your design, you'll begin to see that everything on your page can be "put into a box". Once you identify the overall "box design" of your page, you can create your "boxes", or "content containers" using Block tags (like div), or creating specific classes (like header and footer), using CSS.

BROWSER DEVELOPER TOOLS

When the browser reads an HTML file, it generates a "tree-like structure" referred to as a "Document Object Model". In simpler terms, the browser makes a "mock-up" of your webpage. The difference is the browser's "sketch" is a bit more complex. To view the "DOM" for your webpage, open the "Developer Tools" menu option in your browser. You can use "Developer Tools" to make temporary changes to your webpage without editing the actual HMTL code.

Part 3: Adding Style to Your Webpage with CSS (Cascading Style Sheets)

FORMATTING WITH CSS

While HTML provides the content for a webpage, CSS is what let's you "make it pretty". CSS, while it works with HTML, has it's own language, syntax, and rules. You can link to an external CSS file, incorporate internal CSS styles in the header of an .html page, or create an inline CSS style anywhere on a webpage.

Using CSS decreases the amount of HTML tags needed for a particular look/feel, and helps to avoid repeating tags when you want to re-use a style on multiple parts of the same page. This becomes more and more important as your website becomes larger. Imagine creating a 15-page website with a specific color-scheme for every paragraph of text on each page, and then your client tells you that they want to use a different color-scheme. If you didn't use CSS, you would have to make the color code changes on ALL 15 pages! Fortunately for you, however, you took this course and knew to use CSS. And because you created a class that defined the color-scheme, you only have to change the color code once in the CSS style definition you created...whew!
